# Handbook — people-context

Living documentation for the People Context service — the central identity registry of the ACDG ecosystem.

## Structure
- **domain/** — Core domain definition: aggregates, invariants, integration map.
- **architecture/** — Stack, layers, deployment topology.
- **codebase/** — Internal module contracts and boundaries.
- **principles/** — Design principles and non-negotiable rules.
- **tooling/** — Framework and runtime guides (Bun, Elysia, PostgreSQL).
- **quality/** — Testing strategy, coverage targets, reliability plans.
- **process/** — Operational agreements, versioning, PR flow.
- **cicd/** — Pipelines, environments, release governance.
- **references/** — External links, papers, and standards.
- **reports/** — Chronological records (sessions, reviews, incidents).

## Immediate goals
- Operate as an independent service with clear ownership and low coupling.
- Serve as the minimal identity index for all ACDG systems.
- Keep technical and domain decisions traceable.
